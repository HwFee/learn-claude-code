# s05 学习笔记 —— Skills

## 核心思想

> **"Load knowledge when you need it, not upfront"**
>
> 需要时再加载知识，不要一开始就全部塞进 system prompt。s05 解决的是：**知识文档太多，system prompt 撑爆的问题。**

---

## 问题背景

假设你有 100 个 skill（PDF处理、代码审查、MCP构建...），每个 skill 2000 tokens：

| 方案 | 每次 API 调用消耗 | 问题 |
|------|------------------|------|
| 全部塞进 system prompt | 100 × 2000 = 20万 tokens | 贵死，上下文爆炸 |
| s05 两层注入 | 平时 200 tokens，用到时才 +2000 | 省钱，按需加载 |

---

## 两层注入架构

```
Layer 1 (廉价常驻)              Layer 2 (按需加载)
==================              ==================
System Prompt:                   tool_result:
Skills available:                <skill name="pdf">
  - pdf: Process PDF...           Full PDF processing instructions
  - code-review: Review...        Step 1: ...
  - mcp-builder: Build...         Step 2: ...
                                  </skill>
```

| 层级 | 位置 | 内容 | Token 消耗 |
|------|------|------|-----------|
| Layer 1 | System Prompt | skill 名称 + 简短描述 | ~100 tokens/skill |
| Layer 2 | tool_result | skill 完整 body | 几百到几千 tokens |

## 新增内容（相比 s04）

### 1. SkillLoader 类

```python
class SkillLoader:
    def __init__(self, skills_dir: Path):
        self.skills = {}
        self._load_all()   # 扫描 skills/ 下所有 SKILL.md
```

#### _load_all —— 扫描目录

```python
for f in sorted(self.skills_dir.rglob("SKILL.md")):
    text = f.read_text()
    meta, body = self._parse_frontmatter(text)
    name = meta.get("name", f.parent.name)
    self.skills[name] = {"meta": meta, "body": body, "path": str(f)}
```

- `rglob("SKILL.md")`：递归搜索所有 SKILL.md
- `meta.get("name", f.parent.name)`：优先用 frontmatter 里的 name，没有就用文件夹名

#### _parse_frontmatter —— 解析 YAML 头

```python
def _parse_frontmatter(self, text: str) -> tuple:
    match = re.match(r"^---\n(.*?)\n---\n(.*)", text, re.DOTALL)
    if not match:
        return {}, text          # 没有 frontmatter，整个文件当 body
    meta = yaml.safe_load(match.group(1))
    body = match.group(2).strip()
    return meta, body
```

SKILL.md 格式：
```markdown
---
name: pdf
description: Process PDF files...
---
        ↑ 从这里劈开
# PDF Processing Skill
...body 内容...
```

- `^---\n(.*?)\n---\n(.*)`：正则匹配两个 `---` 之间的内容
- `re.DOTALL`：让 `.` 匹配换行符，使 `.*` 可以跨行
- `(.*?)` 非贪婪匹配：最短匹配第一个 `---` 到第二个 `---` 之间的内容
- 分离成 `meta`（YAML 元信息）和 `body`（Markdown 正文）

**如果文件没有 frontmatter：** 返回 `({}, text)`，meta 是空字典，整个文件当 body

**如果 frontmatter 里没有 `name` 字段：** `_load_all` 里用文件夹名兜底：

```python
name = meta.get("name", f.parent.name)   # 优先 frontmatter 里的 name，没有就用文件夹名
```

#### get_descriptions —— Layer 1

```python
def get_descriptions(self) -> str:
    for name, skill in self.skills.items():
        desc = skill["meta"].get("description", "No description")
        line = f"  - {name}: {desc}"
```

生成 skill 目录列表，注入 system prompt：
```
Skills available:
  - pdf: Process PDF files...
  - code-review: Review code...
```

#### get_content —— Layer 2

```python
def get_content(self, name: str) -> str:
    skill = self.skills.get(name)
    return f'<skill name="{name}">\n{skill["body"]}\n</skill>'
```

返回完整 body，用 XML 标签包裹：
```xml
<skill name="pdf">
# PDF Processing Skill
...完整内容...
</skill>
```

### 2. SYSTEM —— Layer 1 注入

```python
SYSTEM = f"""You are a coding agent at {WORKDIR}.
Use load_skill to access specialized knowledge before tackling unfamiliar topics.

Skills available:
{SKILL_LOADER.get_descriptions()}"""
```

- system prompt 里只有 skill **目录**（名称+描述）
- 告诉模型"遇到不熟悉的话题，先用 load_skill 加载知识"

### 3. load_skill 工具

```python
TOOL_HANDLERS = {
    ...
    "load_skill": lambda **kw: SKILL_LOADER.get_content(kw["name"]),
}
```

```python
{"name": "load_skill", "description": "Load specialized knowledge by name.",
 "input_schema": {"properties": {"name": {"type": "string"}}, "required": ["name"]}}
```

- 参数只有 `name`（skill 名称）
- 返回该 skill 的完整 body

## 交互流程示例

用户：`帮我提取这个 PDF 里的文字`

**Round 1：**
- 模型看到 system prompt 里有 `pdf: Process PDF files...`
- 模型决定先加载 pdf skill
- 工具调用：`load_skill(name="pdf")`

**Round 2：**
- tool_result 返回完整的 pdf skill body（PyMuPDF、pdftotext、ReportLab 等用法）
- 模型现在"学会了"怎么处理 PDF
- 工具调用：`bash → pdftotext input.pdf -`

## 设计哲学

| 要点 | 说明 |
|------|------|
| 目录常驻 | skill 名称和描述放 system prompt，让模型知道"我能做什么" |
| 内容按需 | 完整 skill 只在调用 load_skill 时才注入 |
| 省钱 | 不用的 skill 不消耗 token |
| 可扩展 | 加新 skill = 新建文件夹 + SKILL.md |
| YAML frontmatter | 标准格式，meta 信息和 body 分离 |

## 补充知识点

### System prompt 每次都会重复发送

```python
response = client.messages.create(
    model=MODEL, system=SYSTEM,   # ← 每次循环都传！
    messages=messages,
    tools=TOOLS, max_tokens=8000,
)
```

- Anthropic API 是**无状态**的，每次请求独立
- 模型不会"记住"上一次的 system prompt
- **system prompt 的 tokens 每次都要计费**

**这就是为什么要精简 system prompt：**

假设一次对话 20 轮 API 调用：

| System prompt 大小 | 20 轮总消耗 |
|-------------------|------------|
| 500 tokens（精简） | 500 × 20 = 10,000 |
| 5000 tokens（塞满 skill） | 5000 × 20 = 100,000 |

把 system prompt 从 5000 压到 500，20 轮对话省 90,000 tokens。
