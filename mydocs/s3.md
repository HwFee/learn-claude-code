# s03 学习笔记 —— TodoWrite

## 核心思想

> **"An agent without a plan drifts"**
>
> 没有计划的智能体会漂移。s03 给模型一个 todo 工具，让它自己规划多步骤任务，harness 负责在它忘记时唠叨提醒。

---

## 新增内容（相比 s02）

### 1. TodoManager 类

```python
class TodoManager:
    def __init__(self):
        self.items = []
```

- 纯 Python 类，状态存在**内存**里（程序退出丢失，s07 会持久化到文件）
- 模块级单例：`TODO = TodoManager()`，整个程序共享一个实例

#### update 方法 —— 校验 + 更新

```python
def update(self, items: list) -> str:
    if len(items) > 20:
        raise ValueError("Max 20 todos allowed")
    ...
    if in_progress_count > 1:
        raise ValueError("Only one task can be in_progress at a time")
```

**校验规则：**

| 规则 | 目的 |
|------|------|
| 最多 20 个 | 防止超长列表撑爆上下文 |
| 必须有 text | 防止空任务 |
| 状态只能是 pending/in_progress/completed | 状态机约束 |
| 只能有一个 in_progress | **强制单任务聚焦**，防止同时开多个坑 |

#### render 方法 —— 格式化输出

```python
def render(self) -> str:
    marker = {"pending": "[ ]", "in_progress": "[>]", "completed": "[x]"}[item["status"]]
```

输出格式：
```
[ ] #1: 创建项目目录
[>] #2: 编写主程序
[x] #3: 初始化 git

(1/3 completed)
```

这是**给模型看的状态**，模型根据它决定下一步做什么。

### 2. todo 工具

```python
TOOL_HANDLERS = {
    ...
    "todo": lambda **kw: TODO.update(kw["items"]),
}
```

- 模型调用时传 `{"items": [{"id": "1", "text": "...", "status": "pending"}, ...]}`
- 返回 render() 的字符串，模型能看到完整任务状态

**格式化后的 JSON Schema：**

```json
{
  "name": "todo",
  "description": "Update task list. Track progress on multi-step tasks.",
  "input_schema": {
    "type": "object",
    "properties": {
      "items": {
        "type": "array",
        "items": {
          "type": "object",
          "properties": {
            "id": { "type": "string" },
            "text": { "type": "string" },
            "status": {
              "type": "string",
              "enum": ["pending", "in_progress", "completed"]
            }
          },
          "required": ["id", "text", "status"]
        }
      }
    },
    "required": ["items"]
  }
}
```

**结构解析：**

| 层级 | 含义 |
|------|------|
| `input_schema` | 整个输入是一个对象 |
| `properties.items` | 对象里有一个 `items` 字段 |
| `items.type: "array"` | `items` 是一个数组 |
| `items.items` | 数组的每个元素是... |
| `...type: "object"` | ...一个对象 |
| `...properties.id/text/status` | 对象有三个字段 |
| `...status.enum` | `status` 只能是三种之一 |
| 两个 `required` | 外层必填 `items`，内层必填 `id`+`text`+`status` |

### 3. Nag Reminder（唠叨提醒）

```python
def agent_loop(messages: list):
    rounds_since_todo = 0
    while True:
        ...
        used_todo = False
        for block in response.content:
            if block.type == "tool_use":
                ...
                if block.name == "todo":
                    used_todo = True
        
        rounds_since_todo = 0 if used_todo else rounds_since_todo + 1
        if rounds_since_todo >= 3:
            results.append({"type": "text", "text": "<reminder>Update your todos.</reminder>"})
```

**逻辑：**

| 情况 | rounds_since_todo | 行为 |
|------|------------------|------|
| 本轮调了 todo | 重置为 0 | 正常 |
| 本轮没调 todo | +1 | 检查 |
| 连续 3 轮没调 | >= 3 | **注入提醒** |

**注入的提醒：**
```python
{"type": "text", "text": "<reminder>Update your todos.</reminder>"}
```

- 不是 `tool_result`，而是 `text` 类型——假装是"用户"说的一句话
- harness 不替模型规划，只是**提醒它别忘了更新进度**
- 这是 **harness 对模型的轻量干预**

### 4. System Prompt 更新

```python
SYSTEM = f"""You are a coding agent at {WORKDIR}.
Use the todo tool to plan multi-step tasks. Mark in_progress before starting, completed when done.
Prefer tools over prose."""
```

| 新增指令 | 含义 |
|---------|------|
| `Use the todo tool to plan multi-step tasks` | 告诉模型用 todo 做计划 |
| `Mark in_progress before starting, completed when done` | 明确状态转换规则 |
| `Prefer tools over prose` | 少废话，多调用工具 |

### 5. 错误处理增强

```python
try:
    output = handler(**block.input) if handler else f"Unknown tool: {block.name}"
except Exception as e:
    output = f"Error: {e}"
```

- s03 新增 try/except，handler 抛异常（如 TodoManager 校验失败）不会崩溃，返回错误信息给模型

## 交互流程示例

用户：`帮我写一个计算器脚本并测试`

| 轮次 | 模型调用 | rounds_since_todo | 备注 |
|------|---------|------------------|------|
| 1 | `todo`（创建 3 个任务） | 0 | 初始化计划 |
| 2 | `write_file`（写代码） | 1 | 没调 todo |
| 3 | `write_file`（写测试） | 2 | 没调 todo |
| 4 | `bash`（运行测试） | 3 >= 3 | **注入 `<reminder>Update your todos.</reminder>`** |
| 4 后续 | `todo`（更新所有为 completed） | 0 | 模型收到提醒，更新状态 |
| 5 | `end_turn`（回复用户） | — | 任务完成 |

## 与 s02 的对比

| | s02 | s03 |
|--|-----|-----|
| 规划能力 | 无 | TodoManager + todo 工具 |
| harness 干预 | 无 | Nag Reminder（连续 3 轮未更新则提醒） |
| 错误处理 | 无 try/except | 有 try/except |
| System Prompt | `Act, don't explain` | `Prefer tools over prose` + todo 指令 |
| 状态存储 | 无 | 内存（TodoManager 单例） |

## 设计哲学

| 要点 | 说明 |
|------|------|
| 模型自己规划 | harness 不替模型写计划，只提供工具和提醒 |
| 状态可见 | render() 输出给模型看，也给人看 |
| 轻量干预 | Nag 提醒但不强制执行 |
| 校验约束 | 限制任务数量、单 in_progress、三种状态 |
