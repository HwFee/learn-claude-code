### `_clear_dependency` 解除依赖

```python
def _clear_dependency(self, completed_id: int):
    """Remove completed_id from all other tasks' blockedBy lists."""
    for f in self.dir.glob("task_*.json"):
        task = json.loads(f.read_text())
        if completed_id in task.get("blockedBy", []):
            task["blockedBy"].remove(completed_id)
            self._save(task)
```

## 追问：为什么不直接从文件读 JSON 字符串？

### 用户的疑问

看到这段代码：

```python
def _load(self, task_id: int) -> dict:
    path = self.dir / f"task_{task_id}.json"
    if not path.exists():
        raise ValueError(f"Task {task_id} not found")
    return json.loads(path.read_text())       # 字符串 → 字典

def get(self, task_id: int) -> str:
    return json.dumps(self._load(task_id), indent=2, ensure_ascii=False)   # 字典 → 字符串
```

**问题**：为什么不直接从文件读 JSON 字符串返回？这样不是少了一次 `loads` 又 `dumps` 的来回转换吗？

### 答案：不是不能，而是设计上的取舍

#### 1. `_load` 必须返回 dict，因为内部方法要用

`_load` 是一个**内部辅助方法**，不只是给 `get()` 用的：

```python
def update(self, task_id: int, ...):
    task = self._load(task_id)    # ← 需要 dict 才能修改
    task["status"] = status        # ← 改字典
    task["blockedBy"] = ...        # ← 改字典
    self._save(task)               # ← 保存字典
```

`update()` 必须拿到字典才能修改字段。所以 `_load` 返回 dict 是必要的。

如果 `get()` 绕过 `_load` 直接读文件字符串，那 `_load` 的**文件存在检查**和 **JSON 合法性校验** 就要在 `get()` 里再写一遍，代码重复。

#### 2. `json.dumps()` 保证了输出格式一致

假设直接从文件读字符串：

```python
def get(self, task_id: int) -> str:
    path = self.dir / f"task_{task_id}.json"
    return path.read_text()   # 文件里是什么就返回什么
```

这样会有一个问题：如果有人在文件系统里手动改了 JSON，变成了一行：

```json
{"id":1,"subject":"xxx","status":"pending","blockedBy":[],"owner":""}
```

那模型收到的 tool_result 就是这种**没有换行、没有缩进的紧凑 JSON**，可读性很差。

而通过 `json.dumps(..., indent=2)`，无论文件里格式多乱，返回给模型的永远是**格式化的、带缩进的、人眼友好的 JSON**：

```json
{
  "id": 1,
  "subject": "xxx",
  "status": "pending",
  "blockedBy": [],
  "owner": ""
}
```

#### 3. 性能损耗可以忽略

一次 `loads` + `dumps` 对一个几 KB 的 JSON 文件来说，耗时是**微秒级别**的。这种可读性和代码复用性的收益，远大于这点性能损耗。

### 总结

| 方案 | 优点 | 缺点 |
|------|------|------|
| **当前实现**（`_load` → `get`） | 代码复用、格式统一、有校验 | 多一次 loads+dumps |
| 直接读文件字符串 | 少一次转换 | 格式不可控、校验逻辑要重复写 |

**当前的写法是更优的设计**：牺牲几乎为零的性能，换取代码复用和输出格式的一致性。
